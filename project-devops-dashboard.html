Multi-Account AWS & GitHub DevOps Analytics Platform

AWS Python Boto3 GitHub API Power BI CloudTrail

Project Overview

This project delivers a comprehensive, unified data intelligence solution designed to monitor, audit, and optimize multi-account AWS environments and GitHub CI/CD workflows. By engineering a custom Python ETL pipeline, the platform extracts raw metadata across isolated environments and transforms it into an interactive Power BI suite.

This empowers DevOps leads and stakeholders to easily track cross-account resource consumption, manage CloudTrail logs, monitor security posture, and optimize workflow efficiency from a single pane of glass.

📊 Data Architecture

The platform is built on a star-schema data model integrating AWS multi-account data and GitHub CI/CD metrics into a unified analytical layer.

Data Model Highlights

  • Star-schema design with separate fact tables for AWS services and GitHub workflows
  • Multi-account AWS data consolidated through Python ETL with Boto3
  • CloudTrail logs, billing data, and GitHub API integrated into shared dimension tables
  • Custom DAX measures for cost optimization and CI/CD performance tracking

Key Features & Dashboard Breakdown

DevOps Overview & Security Posture

Tracks active vs. inactive IAM users, total running EC2 instances, and key distribution metrics. Maps S3 bucket capacities and visualizes specific service usage across your infrastructure to spot unused or orphaned resources.

Cost & Log Intelligence

Integrates AWS Billing API metrics with live CloudTrail events. Surfaces the most expensive infrastructure items alongside real-time user action logs, helping match spikes in costs with specific API events.

Resource Monitor & Performance Analytics

Monitors EC2 compute health using historical CPU utilization graphs and localized line metrics. Tracks network activity (NetworkIn / NetworkOut) and provides an Instance Analysis broken down by state (Running/Stopped) and instance types.

GitHub Workflow Analytics

Aggregates repository CI/CD metadata to measure delivery performance. Visualizes workflow attempt volumes, calculates failure/success ratios, and tracks developer-specific workflow metrics to isolate pipeline bottlenecks.

Technical Architecture & Data Engineering

1

Data Orchestration (Python)

Developed a modular script utilizing Python to programmatically extract metadata from two separate AWS accounts and target GitHub repositories.

2

Data Modeling (Power BI Desktop)

Engineered a robust star-schema relational model mapping multiple operational fact tables back to core shared dimensions.

3

DAX & Advanced Analytics

Formulated custom Data Analysis Expressions (DAX) to aggregate cloud spend, calculate rolling network bandwidth totals, and dynamic time-series performance metrics.
